# Design Principles

Core principles that guide all design audit decisions.

## The 8 Principles

### 1. Empty States Are Features
Every empty state needs warmth, action, and context. Never show a blank slate without guidance.

### 2. Every Screen Has Hierarchy
What does the user see first, second, third? If everything competes for attention, nothing wins.

### 3. Specificity Over Vibes
"Clean, modern UI" is not a design decision. Be specific about typography, spacing, and visual weight.

### 4. Edge Cases Are User Experiences
47-character names, zero results, error states — these aren't afterthoughts, they're core experiences.

### 5. AI Slop Is the Enemy
Generic patterns feel AI-generated, not intentional. Every design choice should feel deliberate.

### 6. Responsive Is Not "Stacked on Mobile"
Each viewport gets intentional design. Mobile isn't just desktop columns stacked vertically.

### 7. Accessibility Is Not Optional
Keyboard navigation, screen readers, color contrast, touch targets — these are requirements, not nice-to-haves.

### 8. Subtraction Default
If a UI element doesn't earn its pixels, cut it. Start by removing, not adding.

## Application

When auditing, reference these principles to justify findings:

- "This empty state violates Principle 1 — missing warmth and action"
- "This layout violates Principle 6 — mobile is just stacked desktop"
- "This hierarchy violates Principle 2 — too many elements competing"
