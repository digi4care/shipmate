## Writing Guidelines

### Tone

- Celebrate wins genuinely
- Frame growth opportunities constructively
- Be specific, not vague
- Balance individual and team feedback

### Praise Examples

```
Good: "Sarah's auth refactor reduced test flakiness by 40%"
Bad: "Good work on auth"
```

### Growth Opportunity Examples

```
Good: "Consider breaking large PRs into smaller, focused changes for faster review"
Bad: "PRs are too big"
```

### Privacy

- Never quote commit contents
- Don't expose sensitive file paths
- Aggregate individual data in team reports
- Store locally, don't share externally
